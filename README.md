# Privacy Policy

**Effective Date:** February 26, 2026  
**Last Updated:** February 26, 2026

This Privacy Policy explains how Unreal NodeCode ("we", "our", "us") handles information when you use the Unreal NodeCode app.

## 1. Local-First Data Handling

Unreal NodeCode is currently designed as a local-first app.

- Blueprint/project data is stored on your device.
- App state is persisted locally (for example, app storage used by the app runtime).
- We do not currently run a first-party backend that ingests your project graphs or generated code by default.

## 2. Data We Process

### a) Data stored locally on your device
- Projects, graphs, nodes, variables, enums, structs, interfaces, and generated output metadata.
- Local app preferences and cached catalog/source files.

### b) GitHub integration data (only when you use GitHub features)
- GitHub access token (stored locally on your device to keep you signed in).
- Repository/branch selections and file diffs needed for pull/sync flows.
- File content you explicitly choose to pull/sync through GitHub APIs.

GitHub data is sent to GitHub endpoints as part of user-initiated actions.

### c) External links and forms
If you open external feedback forms, Discord links, or other links, those third-party services may collect information under their own policies.

### d) UE catalog/source fetches
When loading function/type catalogs, the app requests JSON files from public GitHub URLs.

## 3. What We Do Not Currently Collect by Default

In the current app implementation, we do not include first-party analytics or crash SDKs that automatically upload telemetry to our own servers.

If that changes, this policy will be updated before or at rollout.

## 4. How Information Is Used

Information is used to:
- Operate app features locally on your device.
- Execute user-requested GitHub workflows (login, pull, sync, repo/branch operations).
- Load public function/type catalogs from GitHub.
- Support user-requested feedback/reporting workflows via external links.

## 5. Data Sharing

We do not sell personal information.

Data may be shared with third parties only when required by features you use, including:
- GitHub (for auth and repo operations you trigger)
- External websites/forms you open from the app
- Legal authorities if required by law

## 6. Retention

Local app data remains on your device until you remove it (for example by uninstalling the app or clearing app data).

Third-party services (such as GitHub or form providers) retain data according to their own retention policies.

## 7. Security

We apply reasonable safeguards in the app, but no storage or transmission method is perfectly secure.

Because GitHub token storage currently uses local app storage for persistence, treat your device security (passcode, OS updates) as part of your security model.

## 8. Your Choices

You can:
- Avoid using GitHub features if you do not want GitHub data exchange.
- Remove local app data by uninstalling/clearing app storage.
- Log out of GitHub from within the app.
- Avoid opening external links/forms.

## 9. Children's Privacy

The app is not directed to children under 13 (or higher minimum age where required by local law).

## 10. Beta Notice

Unreal NodeCode is in active development. Data flows may evolve as features are added.  
We will update this policy when material changes are introduced.

## 11. Changes to This Policy

We may update this Privacy Policy periodically.  
Updates will be reflected by changing the "Last Updated" date.
